# OSC52 tmux Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make clipboard copy from tmux and nested/remote tmux sessions propagate to the local macOS clipboard via OSC52 while preserving local `pbcopy` behavior.

**Architecture:** Keep the outer/local tmux responsible for local clipboard integration and enable OSC52 passthrough so inner tmux sessions and remote apps can forward clipboard writes upstream. Document the interaction model clearly so nested tmux users know when outer tmux copy mode versus OSC52-driven app yanks are used.

**Tech Stack:** tmux 3.5a, macOS `pbcopy`, OSC52 terminal escape sequences, iTerm2 clipboard support
